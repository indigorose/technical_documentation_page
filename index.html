What is HTML

HTML (Hypertext Markup Language) is the code that is used to structure our web pages and its contents. We can use paragraphs, bullet points, images and data tables within HTML to structure the page.

To help define the structure of our content, HTML operates using a series of elements which we then use to enclose or wrap our content so that it appears in certain ways.

The quick brown fox jumps over the lazy dog

The above is a simple line of content, however by using elements, in this case "p",we can change the meaning within the HTML as a paragraph.

<p>The quick brown fox jumps over the lazy dog</p>
Anatomy of an HTML Element

Following code above, it shows a basic HTML element which can be broken down further following the below picture:

The main parts of the element are as follows:

  1. The opening tag: This consists of the name of the element (in this case p for paragraph), wrapped in the opening and closing angle brackets. This states where the element begins or starts to take effect - in this case where the paragraph begins.
  2. The closing tag: This is the same as the opening tag, except that it included a forward slash before the element name. This tells us when the paragraph ends. Note. Failure to include the proper closing tab can lead to errors and alternative results.
  3. The content: This is the content of the element, which in this example, is just text.
  4. The element: The element comprises of the opening tag, the closing tag and the content.

Attribute

Elements can also have attributes, let's look at example below:

Attributes allow the element to contain extra information that won't appear in the content but can sometimes help with its appearance.

In the above example class is the attribute name and header-text is the attribute value. The class attribute allows us to give the element a non-unique identifier that can be used to target it later work, most notably in css styling.

When using an attribute, it should always have the following:

  1. A space between it and the element name (or the previous attribute if many are being used within the element).
  2. The attribute name followed by the equal sign.
  3. The attribute value is wrapped in quotation marks.

Note. It is highly recommended to use quotation marks for all attribute values even if it is simple, doesn't contain ASCII whitespace or certain characters as it helps the code be consistent and understandable to others.

Nesting & Empty Elements

Nesting Elements

We can also add elements to our element, we call this nesting. For example if we want to stress the word "jumps" in our content by making it bold, we can wrap this word using the <strong> element, which means that the world will be emphasised:

<p>The quick brown fox <strong>jumps<strong> over the lazy dog.</p>

Note. It is important to note the order of your elements to ensure successful nesting. In the above example, we can open with the <p> element and then use the <strong> element but when we close it must be in the reverse order.

Thus the code below is incorrect.

<p> The quick brown fox <strong>jumps</p> over the lazy dog.</strong>

Empty Elements

We can also use empty elements whereby a closing tag is not required. Following the example below:

<img src="HTML_code_pic1.jpeg" alt="HTML code breakdown>

We can note in this example that the above code does not contain a closing </img> tag despite using two attributes. This is because the image element doesn't require a wrap content to affect its place on the page.This tag will embed the image as it appears.

Anatomy of an HTML Document

Following the basics of an HTML element, we can look at their effectiveness when combined. In the example below, we will look at some of the most common elements in an HTML document


    <!DOCTYPE html>
    <html>
    <head>
        <meta charset="UTF-8">
        <title>Technical Document Page</title>
    </head>

    <body>
        <header>Introduction to HTML</header>
    </body>
    
    </html>

Let's take a closer look at the different elements used above:

  • <!DOCTYPE html< - doctype. It is a required preamble. In the infancy of HTML, this element was used to ensure good HTML was used and minimise errors. Nowadays with the advent of more advanced error checking, it helps to make sure the document behaves correctly.
  • <html> </html> - Known as a root element, this is usually found wrapping all the other elements in the document.
  • <head> </head> - The head element is used to wrap elements that will tell us about the page but won't be displayed to the page viewers. Most commonly, this is where you will find the links to CSS style sheet, the page title and textual content direction.
  • <meta charset="UTF-8"> - As mentioned in the head tag, the meta element allows us to set the characters for the document. In many cases using UTF-8 allows for the vast majority of characters written in any language. It helps to have this declared in your HTML document to avoid potential issues later on.
  • <title></title> - this element sets the title of your page. It appears in the browser tab when loaded. It also appears when you bookmark/favourite the page.
  • <body> </body> - Lastly the <body> element. This element will wrap the main content of your HTML to be viewed by the majority of users. Whether it is sound, video, images or text, all content will be here and viewed.

Images & Accessibility

As mentioned in other sections, images can be embedded into our HTML and is usually in the empty element format below:

<img src="tamagotchi_1.png" alt="tamgotchi_1">

Using the src attribute, the image will be embedded directly into our HTML document. Furthermore, we also include an alt attribute with the text "tamagotchi_1". Known as the alternative attribute, it allows us to include a description of the image when it cannot be viewed for the following reasons:

  1. It helps users who may be visually impaired and may use screen reading software that will pick up this information to assist the user.
  2. If an error occurs and the image is not displayed.

It is important to note that with alternative text, the aim is to be descriptive for users. In the above case, "tamagotchi_1" may not be a suitable description for descriptive technologies. A more useful description such as "Original multicolored tamagotchi toy with chain." could be used.

Links

Links are key to a functioning web page, they are the web! By using a simple element <a> (short for anchor) and the attribute href (standing for hypertext reference) with your website address you can create an effective link.

The following is an example of a link in a HTML document:

<a href="https://www.google.com">Google</a>

This example, Google will be the only word displayed on our page, however by wrapping the word in the <a> element and including the href attribute and website, clicking and selecting the word google activates the action to change the web page to the referenced address.

Note. Do not forget to include the http:// or https:// (known as the protocol) to your website link. Failure to do so may lead to errors and alternative websites.

Text Changes

In this last section, we are going to look at how we can change the text within our HTML document. In particular we will look at changing headings, paragraphs and lists.

Headings

There are heading elements within HTML that allows us to create different types of heading and subheadings. The headings element has six variations, <h1>—<h6>, in most cases not all variations are required.

Paragraphs

Referring to our first review of elements, the paragraph element is our primary element to use when showing regular bodies of text in web pages.

Lists

HTML has two variations to show a list within a web page based on what you would like to show. However the two most common forms are as follows:

  1. Unordered lists are used when the order of the list doesn't matter and is usually stylised with bullet points. Use the <ul> element to wrap this version.
  2. Ordered lists are used to when order does matter, using numbers instead of bullet points and use the <ol>element to wrap the list.
Inside each element we use the additional <li> element for each point.

The following is an example shows HTML markup with headings, paragraphs and links:

    <body>
        <h1>Introduction to HTML</h1>
           <p>This page is going to look at the following:<p>
                <ul>
                    <li>headings</li>
                    <li>paragraphs</li>    
                    <li>links</li>
                </ul>
    </body>
